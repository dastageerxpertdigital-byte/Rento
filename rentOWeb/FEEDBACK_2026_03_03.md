# Admin Portal Feedback & Questions (2026-03-03)

This document captures the recent feedback provided after reviewing the current state of the admin portal, along with answers and action items.

## 1. Dashboard Dummy Data
**Feedback:** The dashboard still shows dummy data. Revenue, the "68 percent" metric, and the number of requests are hardcoded.
**Action Item:** Wire up the `KPICard` components and charts on the Dashboard to fetch real aggregated data from Firestore instead of relying on the placeholder hardcoded props.

## 2. Non-Functional Header Buttons
**Feedback:** Every button on the screens' top right is not functional (e.g., "Add New").
**Action Item:** Implement `onClick` handlers for header action buttons across all views (Listings, Users, Packages, etc.) to trigger the respective creation modals or specific actions.

## 3. Adding Gemini Keys
**Question:** How do I add Gemini keys?
**Answer / Action Item:** The Gemini API keys should be added to the portal configuration so the backend can read them. They belong in the **Settings** view under the *API Integrations* section. We need to ensure the Settings form has a field for `config.admin.geminiApiKey` (or similar mapping to Firestore) where you can paste the key, and clicking "Save" stores it securely in the database.

## 4. Feedback & Support vs Help & Support
**Question:** "Contact Management" contains Feedback and Support, while the Sidebar also contains Help and Support. What's the difference?
**Answer:** 
- **Contact Management (Content Tab):** This displays the messages submitted by *end-users* via the user-facing mobile/web apps (e.g., bug reports, feature requests). This is for you to read and resolve user tickets.
- **Sidebar "Help & Support" (Bottom Menu):** This is typically meant for *you* (the admin) if you need support using the admin portal itself, or to view internal admin documentation.
**Action Item:** Clarify the UI naming. We can rename the sidebar item to "Admin Documentation" or remove it if not needed, to avoid confusion with the user tickets view.

## 5. Anonymous Users
**Feedback:** The Users screen shows "Anonymous User" and there is no name written.
**Action Item:** Ensure the table pulls the `displayName` or `fullName` property correctly from the Firestore `users` collection. If a user registered via phone/email and hasn't set a name yet, "Anonymous User" is correct, but we should verify the data mapping logic to ensure existing names aren't being missed.

## 6. Settings Details & General UI
**Feedback:** Google APIs section in settings is not clickable. Remove the iOS force version check (we don't have an iOS app). Help & Support screen is unnecessary.
**Action Item:** Fix the clickability of the Google APIs section. Remove the iOS version fields from Settings. Remove "Help & Support" from the sidebar navigation completely.

## 7. Major Layout & Navigation Changes
**Feedback:**
- Listings and Requests should be merged into a single "Posts" screen with tabs (can stay separate collections, just UI merge).
- "Content" should be divided, with items moved to the sidebar as distinct options.
- Gemini needs a dedicated sidebar option where both Logs and API Keys are managed.
- Notifications need a separate sidebar menu, with a new option to send a notification to a specific user.
- App Banners/Sliders should be their own separate screen (for admin to see and approve requests regarding banners).
**Action Item:** Perform a significant restructuring of the sidebar and views. Break apart the "Content" tab. Merge Listings/Requests. Create dedicated views for Banners, Notifications, and Gemini.
